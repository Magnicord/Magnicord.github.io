Linear Regression: From Principle to Implementation

Introduction

Suppose we have a dataset giving the area and age of some houses, how can we predict future house prices? Now we introduce linear regression to tackle this prediction problem.

Linear regression model assumes that:

price=warea⋅area+wage⋅age+b\textrm{price} = w_{\textrm{area}} \cdot \textrm{area} + w_{\textrm{age}} \cdot \textrm{age} + b price=warea​⋅area+wage​⋅age+b

Example Concepts
area\textrm{area}area, age\textrm{age}age features (a.k.a. inputs)
price\textrm{price}price target (a.k.a. outputs)
wareaw_{\textrm{area}}warea​, wagew_{\textrm{age}}wage​ weights (a.k.a. parameters)
bbb bias (a.k.a. offset, intercept)

Suppose we have nnn examples and ddd features, the model can be generalized into:

y^=w1x1+⋯+wdxd+b=w⊤x+b.\begin{align} \hat{y} & = w_1 x_1 + \cdots + w_d x_{d} + b \\ & = \mathbf{w}^\top \mathbf{x} + b. \end{align} y^​​=w1​x1​+⋯+wd​xd​+b=w⊤x+b.​​

where:

In the formula above, x\mathbf{x}x corresponds to the features of a single example. We can use the design matrix X∈Rn×d\mathbf{X} \in \mathbb{R}^{n \times d}X∈Rn×d to represent our entire dataset of nnn examples.

Task

Our task is to find the best model for predicting yyy given x\mathbf{x}x, that is, to find the best parameters w\mathbf{w}w and bbb.

Before searching for them, we need two more things:

  1. A measure of the quality of some given model (i.e. loss function)
  2. A procedure for updating the model to improve its quality (i.e. optimization algorithm)

Loss Function

Loss functions quantify the distance between the real and predicted values of the target.

For regression problems, the most common loss function is the squared error.

When our prediction for an example iii is y^(i)\hat{y}^{(i)}y^​(i) and the corresponding true label is y(i)y^{(i)}y(i), the squared error is given by:

l(i)(w,b)=12(y^(i)−y(i))2l^{(i)}(\mathbf{w}, b) = \frac{1}{2} \left(\hat{y}^{(i)} - y^{(i)}\right)^2 l(i)(w,b)=21​(y^​(i)−y(i))2

The constant 12\frac{1}{2}21​ makes no real difference but proves to be notationally convenient when taking the derivative of the loss.

Considering entire dataset, we simply average (or equivalently, sum) the losses on the training set:

L(w,b)=1n∑i=1nl(i)(w,b)=1n∑i=1n12(w⊤x(i)+b−y(i))2L(\mathbf{w}, b) =\frac{1}{n}\sum_{i=1}^n l^{(i)}(\mathbf{w}, b) =\frac{1}{n} \sum_{i=1}^n \frac{1}{2}\left(\mathbf{w}^\top \mathbf{x}^{(i)} + b - y^{(i)}\right)^2 L(w,b)=n1​i=1∑n​l(i)(w,b)=n1​i=1∑n​21​(w⊤x(i)+b−y(i))2

When training the model, we seek parameters (w∗,b∗)(\mathbf{w}^*, b^*)(w∗,b∗) that minimize the total loss across all training examples:

w∗,b∗=argmin⁡w,b L(w,b)\mathbf{w}^*, b^* = \operatorname*{argmin}_{\mathbf{w}, b}\ L(\mathbf{w}, b) w∗,b∗=w,bargmin​ L(w,b)

Minibatch Stochastic Gradient Descent

In each iteration ttt:

  1. Randomly sample a minibatch Bt\mathcal{B}_tBt​ consisting of a fixed number ∣B∣|\mathcal{B}|∣B∣ of training examples
  2. Compute the gradient of the average loss on the minibatch (i.e. 1∣B∣∑i∈Bt∇(w,b)l(i)(w,b)\frac{1}{|\mathcal{B}|} \sum_{i \in \mathcal{B}_t} \nabla_{(\mathbf{w},b)} l^{(i)}(\mathbf{w},b)∣B∣1​∑i∈Bt​​∇(w,b)​l(i)(w,b))
  3. Multiply the gradient by a predetermined learning rate (i.e. η\etaη)
  4. Subtract the resulting term from the current parameter values (i.e. (w,b)(\mathbf{w},b)(w,b)).

We can express the update as follows:

(w,b)←(w,b)−η∣B∣∑i∈Bt∇(w,b)l(i)(w,b)(\mathbf{w},b) \leftarrow (\mathbf{w},b) - \frac{\eta}{|\mathcal{B}|} \sum_{i \in \mathcal{B}_t} \nabla_{(\mathbf{w},b)} l^{(i)}(\mathbf{w},b) (w,b)←(w,b)−∣B∣η​i∈Bt​∑​∇(w,b)​l(i)(w,b)

Note
Minibatch size and learning rate are user-defined. Such tunable parameters that are not updated in the training loop are called hyperparameters.

Defining the Model

We can use Pytorch’s predefined layers, which allows us to focus on layers to use instead of the layers implementation.

Now we define net, an instance of Sequential class.

The Sequential class connects multiple layers together. When given input data, the Sequential instance passes the data to the first layer, then uses the output of the first layer as the input of the second layer, and so on.

In this case, our model contains only one layer, so Sequential is not actually needed. However, since almost all the model in the future will be multi-layered, using Sequential here will make you familiar with the “standard pipeline”.

In PyTorch, the fully connected layer is defined in Linear and LazyLinear classes. The latter allows users to specify merely the output dimension, while the former additionally requires the input dimension. For simplicity, we will use LazyLinear layers whenever we can.
